Architecture · MOSS-SoundEffect

RunPod Serverless Worker

Text-to-sound pipeline on RunPod GPU infrastructure — request handling, autoregressive inference, and S3 delivery.

Request Pipeline

Inference Flow

  • Client sends text + optional duration_seconds
  • Handler validates input and converts duration to RVQ tokens (1s ≈ 12.5 tokens)
  • Engine loads model lazily on first request, cached for subsequent requests
  • MOSS-SoundEffect generates tokens; Audio Tokenizer decodes to 24kHz waveform
  • WAV uploaded to S3 — presigned URL returned in response

Resilience

OOM Handling

  • Auto-caps max_new_tokens on ≤24GB GPUs
  • One-shot retry at lower token limit after CUDA OOM
  • Audio tokenizer offloaded to CPU on preprocessing OOM, restored after
  • Model source: RunPod HF cache → local volume → HuggingFace Hub

Persistence

Network Volume

  • Deps installed once to /runpod-volume/moss-sfx/
  • Sentinel file prevents reinstall on subsequent cold starts
  • Handler files refreshed from Docker image on every boot
